title: Errores Frecuentes
---

## Me aparece el error "Certificado no emitido por AC de confianza"

Este error puede ocurrir por 2 motivos:

- Estás utilizando un certificado de producción en un entorno de pruebas. O por el contrario, un certificado de pruebas en un entorno de producción.
- Si creaste un certificado para representar a un tercero, deberás asegurarte de delegar correctamente el CUIT y el computador fiscal, asi mismo asegurarte que el punto de venta esté habilitado para web services

## Me aparece el error "Computador no autorizado a acceder al servicio"

Este error ocurre cuando el computador fiscal no está habilitado para facturar con el certificado emitido. Para corregirlo, deberás seguir los pasos de la sección 4 (Autorizar computador) de <a href="/arca/configuracion/#certificados">nuestro tutorial de certificados</a>

## Me aparece el error "No aparecio CUIT en lista de relaciones"

El CUIT configurado no coincide con el del certificado. Verificá que hayas escrito correctamente el CUIT en los ajustes del plugin.

## Me aparece el error "No se pudo replicar el total de la orden"

Este error sucede cuando el plugin no puede entender los totales de la órden. En este caso deberás contactarte con nosotros para solucionarlo.

## Me aparece el error "Certificado inválido para el entorno de pruebas/producción"

El certificado de ARCA se genera para un entorno en específico. Por lo que no es posible usar el certificado de **Pruebas** en **Producción** ni viceversa. Para corregirlo deberás utilizar el entorno correspondiente al certificado que estés utilizando, esto se puede cambiar en los ajustes del plugin, en el ajuste **Entorno**.
